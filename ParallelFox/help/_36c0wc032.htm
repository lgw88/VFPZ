 worker::StartCriticalSection
Start Critical Section of code. 
Worker.StartCriticalSection(cCriticalSectionName)

Parameters

cCriticalSectionName
Name of critical section to start.
Use a unique (but not random) name for each critical section.

Remarks

This is similar to VFP's SYS(2336) Critical Section support, but that function only works across threads inside a multi-threaded DLL (MTDLL). ParallelFox uses a mutex which is visible to all worker processes.

Example

Worker.StartCriticalSection("CreateMyTable")
If !File("MyTable.dbf")
   Create Table MyTable ...
EndIf
Worker.EndCriticalSection("CreateMyTable")
