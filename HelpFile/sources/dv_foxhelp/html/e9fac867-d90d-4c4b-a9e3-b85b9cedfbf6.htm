Visual FoxPro 9.0 SP2
_PDRIVER System Variable
See Also 
 

 

Included for backward compatibility. Use the TO FILE ASCII argument in REPORT.

This system variable is undocumented in the original VFP9 help file,
but the following documentation was found in the FoxPro for Windows 2.6 help, and may still be valid:

Specifies a printer driver program for character-based reports.

_PDRIVER = expC

Remarks


You can specify a printer driver program for character-based reports by storing the name of the program to _PDRIVER. Character-based reports are created in FoxPro for MS-DOS and can be run in FoxPro for Windows and FoxPro for Macintosh. In FoxPro for Macintosh, a printer driver program isn't necessary to print character-based reports created in FoxPro for MS-DOS.

Storing the name of a printer driver program to _PDRIVER unloads the current printer driver program if one is loaded. When you store a printer driver program name in _PDRIVER, the PDONUNLOAD procedure in the currently loaded printer driver program executes (if it has a PDONUNLOAD procedure). The printer driver you store to _PDRIVER then loads, and its PDONLOAD procedure executes (if it has a PDONLOAD procedure).

When you store a printer driver program to _PDRIVER, the message "Printer driver installed" appears. To suppress this message, issue SET NOTIFY OFF before you store a printer driver program name to _PDRIVER.
