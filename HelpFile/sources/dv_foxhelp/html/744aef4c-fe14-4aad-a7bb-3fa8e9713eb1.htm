Visual FoxPro 9.0 SP2
UpdateCmdDataSource Property
See Also 
 

Specifies the data source used by the UpdateCmd property. Read/write at design time and run time.

CursorAdapter.UpdateCmdDataSource [= cDataSource]

Return Value

cDataSource

Character data type. The cDataSource parameter specifies the data source used by the UpdateCmd property.

Remarks

Applies To: CursorAdapter Class

The following table describes behaviors that apply to UpdateCmdDataSource depending on the value of UpdateCmdDataSourceType.

UpdateCmdDataSourceType UpdateCmdDataSource behavior

Empty ("")

UpdateCmdDataSource is disregarded. Visual FoxPro uses the data source in the CursorAdapter DataSource property.

However, if the cursor associated with the CursorAdapter object is based on an ActiveX Data Object (ADO) RecordSet, Visual FoxPro uses the functionality of the ADO RecordSet to update a record. To use this functionality, make sure the CursorAdapter DataSourceType property is set to "ADO".

"ADO"

UpdateCmdDataSource must be set to a valid ADO Command object. The ActiveConnection property of the ADO Command object must be set to a valid and open ADO Connection object.

Visual FoxPro sets the CommandText property of the ADO Command object to the value of the CursorAdapter UpdateCmd property. Visual FoxPro parses the command for parameters, creates parameter values if found, and sets them in the Command object.

The update operation occurs as:

 Copy Code
CursorAdapter.UpdateCmdDataSource.Execute()

For more information about using an ADO RecordSet, automatic updating, and generating SQL commands automatically, see Data Access Management Using CursorAdapters.
