XML Web Services Foundation Classes
See Also 
 

The XML Web Service Foundation class libraries, _WS3Utils.vcx and _WS3Client.vcx, contain a collection of classes that serve as the Visual FoxPro XML Web service extensions for Microsoft SOAP Toolkit version 3.0. Use these classes to register XML Web services in compliance with those supported by the Toolkit. You can also publish Visual FoxPro COM Servers (.dll files) as XML Web services using these extensions.

Remarks

In the _WS3Utils.vcx class library, the core engine class, _WebServices, communicates directly with the Toolkit. The _WebServices class contains methods for publishing XML Web services, including those that read type library information from your server and those that generate the necessary XML Web service support files, for example, ASP listener and WSDL service description files. You can also register an external XML Web service using the methods provided.

The _WS3Utils.vcx class library contains other support classes for working with XML Web services. These classes are essentially form classes that assist the user in gathering information for registering and publishing XML Web services. They work in conjunction with the _WebServices class.

In the _WS3Client.vcx class library, the core class for using an XML Web service is WSHandler. This class handles all calls to the SOAP Toolkit 3.0 SOAP Client classes that connect to an XML Web service and call operations.

For more information about using foundation classes, see Guidelines for Using Visual FoxPro Foundation Classes.
