Visual FoxPro 9.0 SP2
_PDSETUP System Variable
See Also 
 

 

Included for backward compatibility. Use the TO FILE ASCII argument in REPORT.

This system variable is undocumented in the original VFP9 help file,
but the following documentation was found in the FoxPro for Windows 2.6 help, and may still be valid:

Loads or clears a printer driver setup for character-based reports.

_PDSETUP = expC

Remarks


_PDSETUP loads a printer driver setup or clears the current printer driver setup. A printer driver setup is a combination of settings that include the printer driver program and printing information such as page orientation, default font size and style, margins, and so on. A printer driver setup is used in FoxPro for Windows when printing character-based reports created in FoxPro for MS-DOS. In FoxPro for Macintosh, a printer driver setup isn't necessary to print character-based reports created in FoxPro for MS-DOS.

Printer driver setups are stored in your FOXUSER.DBF resource file and you can interactively create them and assign them names in the Printer Setup Editing dialog.

You can also load or clear a printer driver setup with SET PDSETUP. To load a printer driver setup, store the name of the setup in _PDSETUP. You can clear the current printer driver setup by storing the null string in _PDSETUP. Storing the null string in _PDSETUP also clears the _PDPARMS array from memory.

If the printer driver setup name you store in _PDSETUP does not exist in your resource file, the current printer setup interface application executes. If you are using GENPD.APP, the default printer setup interface application included with FoxPro, the Printer Setup Editing dialog appears so you can create a setup with the name you specified.

When the printer driver setup name you include in expC does not exist in your resource file, you can prevent the printer driver setup application from executing by preceding the setup name with a dash (-).

To load a default printer driver setup when you start FoxPro for MS-DOS, include the following line in your CONFIG.FP file.

PDSETUP = 'setup name' WITH parm list

Be sure to enclose the setup name in quotation marks.
