General Field Type
See Also 
 

To store OLE objects, use the General field type.

The General field contains a ten-byte reference to the actual contents of the field: a spreadsheet, a word processor document, or a picture, created by another application. The actual type and amount of data, however, depends on the Automation server that creates the object and whether you link or embed the OLE object.

If you link an OLE object, your table contains only the reference to the data and to the application that created it. If you embed an OLE object, the table contains a copy of the data as well as a reference to the application that created it. The size of a General field is limited only by the amount of available disk space. For more information about using OLE in your application, see Sharing Information and Adding OLE.

For specifications about the General field type, see Visual FoxPro Data and Field Types.
